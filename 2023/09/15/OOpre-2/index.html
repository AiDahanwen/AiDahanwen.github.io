C语言，面向过程语言，函数为编程单位，程序功能由函数及其调用来实现。
Java语言，面向对象语言，类为编程单位，程序功能由类及其协作来实现。
复杂性控制问题——层次化设计

类

  • 类中应该放哪些属性和方法？
    取决于程序需要这个类来做什么。数据和方法。
  • 属性和方法在类中出现的位置自由，处于同一层次。
    • 方法中定义的变量是局部变量，不是类的属性。而类中声明的变量，是类的属性。
  • Java程序中的主类
    • 一个Java程序中可以含有多个类，其中一个类拥有main方法，类似C的主函数。这个main方法就是程序的入口，拥有main方法的类称为主类。
    • 主类一般不设置属性
    • 将业务类处理交给对象，而在主类中仅仅负责对象的构建和方法的调用。
  • 类的属性和方法遵循业务需要。要从业务角度分析属性的取值范围约束；分析调用参数的范围限制；分析对象状况的可能异常，分析程序执行时可能出现的异常情况。
  • 类成员的可见性：即类中方法能否访问另一个对象或者类中成员的规则。
    • public：任意外部对象均可访问
    • protected：任意本类或子类对象都可访问
    • private：只有本类对象才能访问

构造方法

  • 构造方法是方法的一种，用于初始化对象属性。
  • Java语言默认为每一个类配一个无参数的构造方法
  • 一旦用户自己定义了构造方法，默认的构造方法则失效。
  • 构造方法与普通方法的区别：构造方法不需要返回值，且不加void。

查询方法

用于获得一个对象所管理的私有化属性数据(private修饰)。

修改方法

用于更改对象私有属性数据的方法。务必弄清修改方法成功执行的前提条件和执行效果。

重名方法（重载）

Java中允许在一个类中定义多个重名的方法，但是参数列表必须有差别。注意，无法仅仅通过方法的返回值区分两个重名函数。

  • 构造方法的重载：提供多个重名的构造方法是经常出现的应用场景，用于做不同对象的初始化。

创建对象

  • 不是所有的类都可以创建对象（比如构造函数用private修饰）
  • 对象创建后务必引用。即使用同类型的指针指向新创建的对象

方法调用的参数传递

  • 具体内容 详见last post
  • Java中除了基础类型，所有使用类所声明的变量都是引用（即：变量的名称储存变量本身的首地址，变量名称指向变量本身）
  • Java中使用任意类型声明的数组变量也是引用
  • 参数调用时，基础类型直接传递值，引用类型传递引用本身（传递地址）。

对象容器

是Java提供的一整套解决方案。用来管理数量不定且动态变化的一组对象。提供了灵活的遍历和插入方法，比自定义的数组更加便于使用。常见的容器有ArrayList和HashMap

ArrayList

  • ArrayList相当于不定长数组，可以根据需要在运行期自动扩展长度。
  • Arraylist表明容器中只能存放bottle类型的对象。
  • ArrayList存储对象引用（地址），而不是对象值.
  • Arraylist提供插入、检索、克隆和删除操作。 示例：

HashMap，散列表

  • 散列表存储键值对(key-value)映射，key与value均为对象引用（地址）。

  • ArrayList中的对象有序排放，但是HashMap无序，且不会记录插入的顺序。因此只能通过key对象映射获得相应的value对象。

单元测试

类是Java的编程和测试单元。

  • 测试是发现程序bug的工程化方法。

    • 静态测试：检查代码逻辑发现问题
    • 动态测试，执行代码发现问题

    单元测试是一种动态测试 。

  • 测试和调试是不同的概念。

    • 测试的目的是发现bug，调试的目的是定位和修复bug
    • 测试——设计输入数据；调试——设计断点

Junit-单元测试方法和工具

  • 可实现对测试用例的管理
  • 断言(assert)：用于检查被测方法的返回值是否符合预期
  • 异常检查：可以检查被测方法是否会抛出预期的异常。
  • 测试流程管理

  • 在现有目录中初始化仓库：在项目目录中进入git并输入

  • 查看提交历史

    • git log 本身会按提交时间列出所有的更新，最近的更新排在最上面。
    • git log -p 用来显示每次提交的内容差异，添加 -2 来要求显示最近两次提交。
    • git log --stat 用来显示每次提交的简略的统计信息。
    • git log --pretty=oneline 表示将每个提交放在一行显示
    • git log --pretty=format:"<keys>" 定制想要显示的记录格式（例如：%h -提交对象的简短哈希串，%an -作者名字，%ar -作者修订日期，%s -提交说明）
    • git log --pretty=format:"<keys>" --graph 添加ASCII字符串形象地展示分支和合并历史。
    • git log --since=2.weeks 显示最近两周内的提交。

  • 撤销操作

    • 使用git commit --amend 重新提交，该命令会将暂存区的文件提交（在漏掉有文件没有提交时）
    • git rest HEAD <file_name> 取消文件暂存
    • git checkout -- <file_name> 撤销对文件的修改，还原为上次提交时的样子。

  • 总结一下重置操作：

    • HEAD:上一次提交的快照；Index：预期的下一次提交的快照，也即暂存区；Working Directory：工作目录。
    • reset与checkout最大的区别：reset移动HEAD所指向的分支（也即HEAD不会改变其指向的分支）；checkout移动HEAD本身。
    • --soft只移动HEAD指向的分支，而不改变Index和HEAD里面的内容。
    • --hard不仅移动分支，还会使HEAD,Index，Working Directory 三部分中的内容一致.
    • git checkout -- [filename]可将工作区的文件filename恢复为暂存区中的状态，该状态不改变HEAD.
